# nemotron-worker

Public Docker image for the [eased](https://github.com/your-org/eased) GPU worker.
Bakes **Nvidia Nemotron Nano 12B VL BF16** weights into the image so vast.ai instances
need no separate model download on cold start.

Published to: `ghcr.io/your-org/nemotron:latest`

---

## Image contents

| Layer | Size |
|---|---|
| vLLM base (`public.ecr.aws/deep-learning-containers/vllm:0.19.0-gpu-py312-cu129-ubuntu22.04-ec2`) | ~15 GB |
| Nemotron Nano 12B VL BF16 weights (`nvidia/NVIDIA-Nemotron-Nano-12B-v2-VL-BF16`) | ~24 GB |
| **Total** | **~40 GB** |

## Runtime environment variables

These are injected by the eased orchestrator at launch time via vast.ai:

| Variable | Required | Default | Description |
|---|---|---|---|
| `VLLM_API_KEY` | Yes | — | Per-instance Bearer token generated by the orchestrator |
| `MODEL_PATH` | No | `/root/.cache/huggingface/nemotron-nano-12b-vl-bf16` | Path to model weights |
| `VLLM_MAX_MODEL_LEN` | No | `4096` | Maximum sequence length in tokens |
| `VLLM_GPU_MEMORY_UTILIZATION` | No | `0.95` | Fraction of GPU VRAM to use |

## Building locally

```bash
DOCKER_BUILDKIT=1 docker build \
  --secret id=hf_token,env=HF_TOKEN \
  -t ghcr.io/your-org/nemotron:latest \
  .
```

`HF_TOKEN` must have read access to the gated
[nvidia/NVIDIA-Nemotron-Nano-12B-v2-VL-BF16](https://huggingface.co/nvidia/NVIDIA-Nemotron-Nano-12B-v2-VL-BF16)
model. It is passed as a BuildKit secret and is **never** written into any image layer.

## GitHub Actions

The `build.yml` workflow triggers on every push to `main` that touches the
`Dockerfile` or `entrypoint.sh`. It runs on standard `ubuntu-latest` runners
(free for public repos) and uses `jlumbroso/free-disk-space` to reclaim ~40 GB
before the build, giving ~54 GB of working space.

**Required repository secret:**

| Secret | Description |
|---|---|
| `HF_TOKEN` | HuggingFace token with access to the gated Nemotron model |
